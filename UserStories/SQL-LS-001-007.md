### EPIC ID: SQL-LS-001

**Name**: Learning spaces management

**Description**: As part of the core features of ThemePark@UCR, the system must allow users to explore and manipulate data related to learning spaces. This includes the ability to list available spaces, view details of a specific space, update existing records, create new entries, and retrieve relevant information.

---

### User Story

```gherkin
Feature: Create learning spaces using a relational database and internal keys

As an administrator user  
I want to be able to create learning spaces using defined specifications  
So that the system accurately reflects the physical layout of the institution while maintaining a normalized and scalable data structure.
```

---

### Acceptance Criteria

```gherkin
  Scenario: Create a new learning space with all specifications and internal ID
Given I am on the administrative module of the system
When I submit the required data to create a learning space (type, height, width, length)
Then the system generates a unique internal ID and successfully registers the space
```

```gherkin
  Scenario: Missing required fields when creating a learning space
Given I am on the administrative module and attempt to create a learning space with incomplete data (e.g., missing type or dimensions)
When I submit the form
Then the system displays an error message indicating which fields are required and does not register the space
```

```gherkin
 Supports internal ID management, along with type, height, width, and length for each learning space

```

### **Non-functional Requirements – View Registered People**

| Category         | Description |
|------------------|-------------|
| **Performance**  | The system should create a learning space within 1 second under normal load. |
| **Usability**    | The form for learning space creation must have clear labels and validations for each field. |
| **Maintainability** | The logic for handling learning space data should be modular and easy to extend (e.g., add new attributes). |
| **Reliability**  | Learning spaces should be saved and retrieved consistently with 99.9% accuracy. |
