### EPIC ID: CPD-LC-001
**Name**: Learning components
**Description**: This epic is about adding components to learning spaces in ThemePark@UCR, like classrooms and labs. These components can be things like whiteboards, chairs, or projectors that users can interact with, so that learning feels more active and engaging. It also focuses on building the functionality to create, read, update, and delete (CRUD) these objects in the system.

---

### User Story
```gherkin
Feature: Automatic ID generation

As an administrator
I want the component IDs to be generated automatically
So I don’t need to worry about choosing or remembering unique IDs when creating components.
```
---

### Acceptance Criteria
```gherkin
Scenario: Component is assigned an auto-generated ID
Given the administrator is creating a new component
When the form is submitted without specifying an ID
Then the system generates a unique ID and saves it with the component
```

```gherkin
Scenario: IDs are unique across all components
Given multiple components are created
When each one is saved
Then each receives a different, unique ID
```
---
### **Non-functional Requirements**

| Category            | Description                                                                  |
| ------------------- | ---------------------------------------------------------------------------- |
| **Security**        | Only when the component is created, the automatic ID is generated.      |
| **Consistency**     | Generated IDs must be globally unique and consistent across the system.      |
| **Usability**       | The UI should clearly indicate that the ID was auto-generated and show the value. |
| **Maintainability** | The ID generation logic should be extensible to accommodate future rules (e.g., custom prefixes or suffixes). |
