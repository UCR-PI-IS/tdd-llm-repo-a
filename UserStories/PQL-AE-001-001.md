### EPIC ID: PQL-AE-001

**Name**: Add Building

**Description**: As part of the core features of ThemePark@UCR, the system must allow administrators to register new buildings in the system to keep track of the university's infrastructure up to date, ensuring no duplicated and access restricted to admin users only.

---

### User Story

```gherkin
Feature: Add and register a new building in the system.

As a ThemePark@UCR Administrator,
I want to be able to register new buildings in the system,
so that I can maintain an up-to-date record of the university’s infrastructure.
```

---
### Acceptance Criteria

```gherkin
  Scenario: Successful adding of the new building and registering of its information 
  Given the administrator added a new building
  When the administrator submits the building's information
  Then the system should store the new building as a new entry in the Buildings Data Base
  And each entry should at least include basic information about the building(e.g. name, size)
```

```gherkin
  Scenario: Prevent addition of new buildings with incorrect property format
  Given the administrator tries to add a new building with incorrect property format
  When the administrator submits the building's information
  Then the system should reject the registration and display an error message
  And the building should not be added to the system.
```

```gherkin
  Scenario: Prevent duplicate building registration
  Given a building with the same name or ID that already exists in the system
  When the administrator attempts to register a new building with the same name or ID
  Then the system should reject the registration and display an error message
  And the building should not be added to the system.
```
