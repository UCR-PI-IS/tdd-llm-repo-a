### EPIC ID: CPD-LC-001
**Name**: Learning components
**Description**:  This epic focuses on enhancing learning spaces within ThemePark@UCR, such as classrooms and laboratories, by adding interactive components like whiteboards and projectors. These components are designed to create an interactive and dynamic learning environment.

---
### User Story

```gherkin
Feature: Create whiteboard in a learning space

As an user  
I want to create a whiteboard  
So that I can display them in a learning space

```
---

### Acceptance Criteria
```gherkin
Scenario: Successfully create a whiteboard in a learning space
Given the user is logged in  
And there are available whiteboards  
And the selected whiteboard fits in the chosen learning space  
When the user creates a whiteboard  
Then the system adds the whiteboard to the chosen classroom or laboratory  
And confirms the addition to the user
```

```gherkin
Scenario: Create a whiteboard with invalid input
Given a user  
When the user submits invalid or incomplete data  
Then the system prevents creation  
And displays an error message indicating the input issue
```

```gherkin
Scenario: Attempt to create a whiteboard that doesn’t fit in the learning space
Given the user is logged in  
And a whiteboard is selected that exceeds the available space dimensions  
When the user tries to create the whiteboard  
Then the system shows an error message indicating that the whiteboard doesn't fit in the space
```


#### Non-funtional requirements

| Category         | Description |
|------------------|-------------|
| **Performance**  | The system must process the creation of a whiteboard in under 2 seconds. |
| **Reliability** | The system must behave consistently under valid conditions and handle errors gracefully when unexpected input or failures occur, ensuring predictable and stable behavior. |
| **Usability**     | The creation form must be clear and user-friendly, providing visible validation and understandable error messages. |
| **Maintainability**    | The implementation must follow Clean Code principles, be modular, and documented to facilitate future maintenance. |
| **Portability** | The functionality must run successfully in the current development and test environments used in this sprint. |
