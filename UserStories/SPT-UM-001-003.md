### EPIC ID: SPT-UM-001
**Name:** User Management
**Description:** As part of the core features of ThemePark@UCR, the system must allow the management of user accounts. This includes the ability to create, view, modify, delete, and retrieve user information. Proper user management is essential for controlling access and customizing the environment.

---
### User Story
```Gherkin
Feature: Create a person

As a user
I want to create a new person in the system
So that I can access the platform
```

---

### Acceptance Criteria
```Gherkin
Scenario: Successful person creation
Given valid person data (e.g., email)
When the create person function is called
Then the system should save a new person record in the database
And should return a success response
```

```Gherkin
Scenario: Reject creation if the person already exists
Given that a person with the same email or identity number already exists in the system
When the create person function is called
Then the system should not create a new person
And should return an error indicating duplication
```
```Gherkin 
Scenario: Reject creation if required fields are missing
Given the person data is missing required fields (e.g., no email)
When the create person function is called
Then the system should return a validation error
And not save anything to the database
```

#### Non-funtional requirements

| Category         | Description |
|------------------|-------------|
| **Performance**  | Person creation should not take longer than 2 seconds under normal system load. |
| **Reliability** | The person creation feature should succeed in 99.9% of cases when provided with valid data. |
| **Usability**     | The person creation form must display clear validation messages. |
| **Maintainability**    | The person creation logic should be modular to support future updates, such as validation rule changes or user model adjustments. |
| **Portability** | The person creation module must work across different deployment environments. |
