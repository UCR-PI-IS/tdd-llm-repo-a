### EPIC ID: PQL-AE-001

**Name**: Add University

**Description**: As part of the core features of ThemePark@UCR, the system must allow administrators to register new universities in the system to keep track of which universities are available for view in the project.

---

### User Story

```gherkin
Feature: Add and register a new university in the system.

As a ThemePark@UCR Administrator,
I want to be able to register a new university in the system,
so that I can maintain an updated record of the universities available in the system.
```

---
### Acceptance Criteria

```gherkin
  Scenario: Successful adding of the new university and registering of its information 
  Given the administrator added a new university
  When the administrator submits the university's information
  Then the system should store the new university as a new entry in the University Data Base
  And each entry should at least include basic information about the university(e.g. name, country)
```

```gherkin
  Scenario: Prevent addition of new university with incorrect property format
  Given the administrator tries to add a new university with incorrect property format
  When the administrator submits the university's information
  Then the system should reject the registration and display an error message
  And the university should not be added to the system.
```

```gherkin
  Scenario: Prevent duplicate university registration
  Given a university with the same name or ID that already exists in the system
  When the administrator attempts to register a new university with the same name or ID
  Then the system should reject the registration and display an error message
  And the university should not be added to the system.
```
